# Security policy

## Supported versions

Narramorph is an unreleased alpha. Security fixes are made on the current `main` branch only; older commits, feature branches, local modifications, and archived artifacts are not supported versions. There is no production service or deployed release at this time.

## Report a vulnerability privately

Use GitHub's [private vulnerability report](https://github.com/zekusmaximus/Narramorph/security/advisories/new). Do not disclose a suspected vulnerability in a public issue, discussion, pull request, or commit.

Include, when available:

- the affected commit, dependency, file, or feature;
- reproduction steps or a minimal proof of concept;
- the security impact and required preconditions;
- whether story content, local reader data, build credentials, or the dependency chain is affected; and
- any suggested mitigation.

Never include real credentials, private reader data, or harmful payloads. Use synthetic values and the least invasive reproduction possible.

The maintainer aims to acknowledge a report within three business days and provide an initial assessment within fourteen calendar days. Remediation and disclosure timing depend on severity and fix complexity. Please allow coordinated remediation before public disclosure.

General defects without a security impact belong in the public issue tracker.
